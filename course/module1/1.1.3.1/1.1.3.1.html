Configuración de parámetros básicos del router

Los routers y switches Cisco tienen muchas similitudes. Admiten sistemas operativos modales y estructuras de comandos similares, así como muchos de los mismos comandos. Además, los pasos de configuración inicial son similares para ambos dispositivos.

Por ejemplo, las siguientes tareas de configuración se deben realizar en todo momento:

  • Asignar un nombre al dispositivo: (Figura 1) para distinguirlo de otros routers.

  • Asegurar acceso de administración: (Figura 2) asegura EXEC privilegiado, EXEC de usuario y acceso remoto.

  • Configurar un aviso: (Figura 3) para proporcionar notificaciones legales de acceso no autorizado.

Guarde siempre los cambios en un router como se muestra en la Figura 4 y verifique la configuración básica y las operaciones del router.

Utilice el verificador de sintaxis de la figura 5 para configurar el router R2.

La Figura uno muestra el comando utilizado para configurar un nombre de host en un router. Los comandos empiezan con el modo EXEC del usuario. El primer comando es configure terminal y este comando pasa al modo de configuración global. El siguiente comando es hostname R1. El router ahora tiene configurado un nombre de host. La figura dos configura el router para que exija contraseñas para el acceso. Los comandos comienzan en el modo de configuración global. El primer comando establece una contraseña al acceder al modo de habilitación en el router, y es enable secret class. La palabra clave secret le indica al router que cifre la contraseña en el archivo de configuración, y la contraseña utilizada es class. Luego pasa al modo de configuración de consola con el comando line console 0. Una vez en el modo de configuración de consola, el comando para establecer una contraseña es password cisco, y cisco es la contraseña real. El siguiente comando indica al router que se requiere la contraseña para iniciar sesión y el comando que se usa para ello es login. Luego pasa al modo de configuración de líneas terminales con el comando line v.t.y. cero espacio 4. Esto restringe el acceso telnet en cinco conexiones. Una vez en el modo de configuración de línea terminal, el comando para establecer una contraseña es password cisco, y cisco es la contraseña real. El siguiente comando indica al router que se requiere la contraseña para iniciar sesión y el comando que se usa para ello es login. El siguiente comando le indica al router que cifre todas las contraseñas en el archivo de configuración. El comando se ejecuta desde el modo de configuración global y es service password guión encryption. La figura tres configura un mensaje de aviso que el router muestra cuando un usuario inicia sesión. El comando se ejecuta desde el modo de configuración global y es banner motd símbolo de dólar Authorized Access Only! símbolo de dólar (Solo acceso autorizado). El símbolo de dólar se utiliza para que el router sepa dónde comienza y dónde finaliza el mensaje de aviso. El símbolo puede ser cualquier símbolo, pero no puede estar en el mensaje. La figura cuatro muestra el comando que se utiliza para guardar el archivo de configuración de RAM a N V RAM. El comando se ejecuta desde el modo EXEC privilegiado y es copy running guión configuration startup guión configuration. La figura cinco es una actividad interactiva que permite que el estudiante configure un nombre de host, restrinja el acceso mediante contraseñas y configure un aviso en un router.
