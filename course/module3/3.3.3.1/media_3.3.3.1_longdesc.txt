La figura muestra un gráfico de flujo para las rutas coincidentes. En la figura, si la ruta final de nivel uno es la mejor coincidencia, el paquete se enruta siguiendo la dirección IP del siguiente salto o la interfaz de salida. La figura dos muestra el proceso si la mejor coincidencia para un paquete es una ruta principal de nivel uno.  Si se encuentra una coincidencia con una ruta secundaria de nivel 2, el paquete se enruta siguiendo la dirección IP del siguiente salto o la interfaz de salida.  La figura tres muestra el proceso si no se encuentra una coincidencia con una ruta secundaria de nivel 2, que es buscar las rutas de superred, incluida la ruta predeterminada, en la tabla de routing.  Si no se encuentra una coincidencia, el paquete se descarta.
