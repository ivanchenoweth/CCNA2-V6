Seguridad de puertos: configuración

En la figura 1, se resume la configuración predeterminada de seguridad de puerto en un switch Cisco Catalyst.

En la figura 2, se muestran los comandos de CLI de Cisco IOS necesarios para configurar la seguridad de puertos en el puerto Fast Ethernet F0/18 del switch S1. Observe que el ejemplo no especifica un modo de violación. En este ejemplo, el modo de violación es shutdown, el modo predeterminado.

En la figura 3, se muestra cómo habilitar las direcciones MAC seguras persistentes para la seguridad de puertos en el puerto Fast Ethernet 0/19 del switch S1. Como se mencionó anteriormente, la cantidad máxima de direcciones MAC seguras se puede configurar de forma manual. En este ejemplo, la sintaxis del comando de Cisco IOS se utiliza para establecer en 10 la cantidad máxima de direcciones MAC para el puerto 0/19. De manera predeterminada, el modo de violación se establece en shutdown.

La figura uno resume la configuración predeterminada de seguridad de puertos de un switch Cisco Catalyst. Se deshabilita la seguridad de puerto, cuando se establece la seguridad de puerto la cantidad máxima de direcciones MAC seguras se establece como uno, se desactiva el modo de violación, y se desactiva el aprendizaje de direcciones MAC persistentes. La figura dos muestra los comandos de CLI necesarios para configurar la seguridad de puertos en una interfaz de puerto de switch. El primer comando es interface fast Ethernet cero barra 18. Este comando pasa del modo de configuración global a una interfaz específica. El siguiente comando es switchport mode access, que convierte al puerto rigurosamente en un puerto de acceso para un terminal. El último comando es switchport port guión security. La figura tres continúa con el ejemplo anterior y agrega el comando switchport port guión security maximum 50. El último comando es switchport port guión security mac guión address sticky.
